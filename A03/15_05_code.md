# RACETRACK – Prototype

## Beschreibung
Ein einfacher Musik-Step-Sequencer für den Adafruit PyBadge.

## Funktionen
- Circle-of-Fifths Musiksystem (F, C, G, D, A, E, B)
- Schrittbasierte Sequenz (Track-System)
- BPM-Steuerung mit Buttons (A = langsamer, B = schneller)
- Display zeigt Note, BPM und Track-Status
- WAV-Audio wird pro Note abgespielt

## Status
- Sequencer, Display und BPM funktionieren
- Audio-Wiedergabe funktioniert aktuell nicht zuverlässig

## Nächste Schritte
- Audio-Problem (WAV / Hardware / CircuitPython) debuggen und am WE lösen


## Aktuelle Einschränkungen (Arduino Nano + PyBadge Ansatz)

- ❌ Arduino Nano kann keine Audioverarbeitung oder WAV-Wiedergabe übernehmen  
- ❌ PyBadge unterstützt nur einen einzigen Audio-Stream gleichzeitig (keine echte gleichzeitige WAV-Wiedergabe)  
- ❌ Mehrschichtiger Sound (Kick + Hi-Hat + Bass gleichzeitig) ist nicht nativ möglich  
- ❌ Arduino Audio-Libraries erlauben in der Regel nur eine aktive Sound-Wiedergabe zur gleichen Zeit  
- ❌ Echtzeit-Audio-Mixing ist ohne spezielle Hardware nicht realisierbar  

---

## Lösung?

Für ein zuverlässiges Mehrspur-Audiosystem wird eine **separate Audio-Engine benötigt**.

Ohne eine dedizierte Audio-Engine gilt:
- Nur einzelne Sounds können abgespielt werden  
- Polyphone oder Drum-Machine-artige Strukturen sind nicht korrekt umsetzbar
